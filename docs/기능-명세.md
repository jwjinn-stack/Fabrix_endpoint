## 1. 개요

FABRIX Endpoint 는 사내 추론 플랫폼(NVIDIA Dynamo + vLLM)을 운영자가 한 곳에서 **관제·평가·배포·통제**할 수 있는 콘솔입니다. 크게 5개 기능군으로 나뉩니다.

1. **관측(Observability)** — 트래픽·품질·GPU·사용량 실시간 관제
2. **거버넌스(Governance)** — 가드레일(PII/jailbreak) 차단·증적·정책, WORM 불변보존
3. **모델 수명주기(Model Lifecycle)** — 레지스트리(Harbor) 임포트 → 엔드포인트 배포 → 추론
4. **검증(Validation)** — 플레이그라운드 채팅, 멀티모델 비교, LLM-as-judge 평가
5. **운영 관리(Admin)** — API 키·앱, 조직·귀속, 사용자(RBAC), 서드파티 자격증명

## 3. 기능 영역별 상세

### 3.1 관제 (Dashboard)
- 기간(1h/6h/24h/7d) 선택에 따른 **트래픽**(QPS·동시처리·성공률), **품질**(TTFT p50/p95·ITL·캐시 적중률), **가드레일**(차단·PII·jailbreak·flag), **GPU**(사용률·KV캐시·MIG효율) 카드.
- **부서·앱별 사용 비중**, **알람** 목록, **시계열 차트**(QPS·TTFT·차단).
- 카드 클릭 → 증적/GPU 상세로 이동(드릴다운).
- *v0.16.0 IA: 관제는 **스크롤 없는 글랜스 화면**으로 유지. 추론 지연 3분할·스케줄러·토큰 분해·Top5 등 분석 심화 지표는 **사용량 리포트로 이동**(§3.14).*

### 3.2 사용량 (Usage)
- `group_by`(모델/부서/앱/키)로 요청 수·프롬프트/완료 토큰·TTFT·ITL 집계 표 + 점유율 스택바 + CSV 내보내기.
- ClickHouse 사용량 롤업 기반(미구성 시 빈 상태).
- *v0.16.0: **추론 지연 3분할·엔진 스케줄러·토큰 분해·Top5 랭킹·추세 forecast** 가 여기로 배치(§3.14) — 분석 심화 화면.*

### 3.3 가드레일 (Guard) — 개요 / 증적 / 정책
- **개요**: 검사·차단·PII·jailbreak·flag 요약, WORM 보존 상태.
- **증적**: 기간·판정(allowed/blocked/flagged)·유형(PII/jailbreak) 필터, 이벤트 표, **상세 모달**(trace·부서·앱·키·정책버전·마스킹 샘플). 변조 불가(ClickHouse + MinIO WORM).
- **정책**: PII/jailbreak/secrets 규칙 enable + action(block/flag) 편집(저장 시 정책버전 반영).
- **분류 테스트**: 임의 텍스트를 Semantic Router로 즉시 분류해 결과 확인.

### 3.4 모델 (Models) — Harbor 레지스트리
- Harbor 에 저장된 모델 카드(프로젝트·태그·크기·pull 수·레퍼런스). 검색.
- 카드 → **엔드포인트 생성**(해당 모델로 위저드 진입) / 상세(슬라이드 패널).
- "Dynamo 가 Harbor 에서 pull 해 서빙" 개념의 저장소 화면.

### 3.5 모델 임포트 (ModelImport) *(신규)*
- 소스 카드 3종: **NVIDIA NGC** / **Hugging Face** / **직접 업로드**.
- 자격증명 상태 칩(HF 토큰/NGC 키 설정 여부)과 [자격증명 관리] 바로가기.
- 모델 ID·Harbor 프로젝트 입력 → **미리보기**(k8s Job 매니페스트 + CLI 대안) → **임포트 잡 실행**.
- 잡: HF에서 다운로드(저장된 `HF_TOKEN` 사용) → 패키징 → `oras push` 로 Harbor 저장 → 모델 화면 표시.
- *검증됨: HF(tiny-gpt2) → Harbor → 모델 화면 표시.*

### 3.6 플레이그라운드 (Playground)
- 서빙 모델 선택 후 채팅. **max_tokens·temperature** 조절.
- 응답별 **지표**(프롬프트/완료 토큰·지연 ms·TPS)와 **가드 배지**(클린 통과/PII 차단/jailbreak 차단).
- **모델 비교**: 동일 입력을 채팅 가능한 모든 모델에 동시 전송해 표 비교.
- **코드 보기**: 선택 모델 호출 스니펫(curl/python/js, 자사 엔드포인트 URL 자동 주입).

### 3.7 평가 (Eval)
- 대상 모델 + 심판(judge) 모델 + 프롬프트/기준 입력 → **LLM-as-judge** 점수(0~5, 0=차단)·근거·지연.
- 가드레일 결과 동반 표시.

### 3.8 엔드포인트 (Endpoints) — 모델 배포
- 배포된 DynamoGraphDeployment 목록(이름·ns·백엔드·replica·귀속·상태·관리주체).
- **생성 위저드**: 목적 프리셋(빠른응답/높은처리량/최소구성), 패턴(agg/agg_router/disagg), 모델 또는 **Harbor 참조**, replica/GPU/max_model_len, 앱·부서 귀속, **추론 접근 방식(ClusterIP/NodePort)**.
- **미리보기**(서버 dry-run 검증 + 매니페스트) → **생성 적용**.
- 삭제는 **FABRIX 생성분만**(운영 리소스 보호). 생성 시 추론 API용 `<name>-api` 서비스 동반 생성.
- *검증됨: Harbor 참조 배포 → 추론 응답 정상.*

### 3.9 GPU/MIG (Gpu)
- 호스트·GPU별 util·메모리(사용/전체)·온도·전력·SM/Tensor activity·MIG 효율. 요약(총 GPU·평균 util/mem·총 전력).

### 3.10 키·앱 (Keys)
- API 키 **발급**(평문 1회 노출)·**폐기**, 앱·이름·모델 스코프·쿼터(RPM/TPD).
- 키별 사용량(요청·토큰) 동반. `x-api-key-id` 헤더로 쿼터·귀속 적용.

### 3.11 조직·귀속 (Organization) *(신규, 검증 진행 중)*
- 부서 → 앱 → 키 + 부서별 사용자의 귀속 트리 조회, 앱의 소속 부서 설정.
- *현재 `/api/v1/org` 런타임 점검 중(확인 필요).*

### 3.12 트래픽 (Traffic)
- 프록시 실측 통계: 총/차단/허용, 차단율, 평균 가드 지연·업스트림 지연·p95, 오버헤드%, 모델별 분포, 분당 요청(QPM). (인메모리 윈도우)

### 3.13 설정 / 서드파티 자격증명 (Settings / Credentials)
- **설정(사용자)**: 사용자·역할(Admin/User/Super)·부서 매핑(RBAC), 인라인 역할 변경, 추가/삭제.
- **서드파티 자격증명** *(신규)*: **HF Model Hub 토큰** + **NVIDIA NGC 키** 등록/수정. 값은 k8s Secret(`fabrix-thirdparty`)에 저장, 화면엔 **마스킹**(예 `hf_****vQF`). 모델 임포트 다운로드에 사용. *검증됨: 저장→마스킹→읽기 왕복.*

### 3.14 v0.16.0 보강 — 추론 성능·관측 심화 (Phase 1 마감)
> 삼성증권 "서빙 모델 성능 모니터링 / 사용자별 사용량 추적" 요구 대응. **추론 성능 분석 지표는 관제(글랜스) 대신 사용량 리포트에 배치**(관제는 한 화면 글랜스 유지).
- **추론 지연 3분할**: TTFT(첫 토큰)·TPOT(토큰당)·E2E(전체) 각 p50/p95/p99 게이지 — 사용량 화면. (출처 dynamo_frontend 히스토그램)
- **엔진 스케줄러**: 실행/대기 요청·큐 대기 p95·KV 캐시 점유. (vllm:num_requests_*, request_plane_queue)
- **토큰 분해**: 입력/캐시 적중/출력 누적 — 캐시 적중률 가시화.
- **Top5 엔드포인트/API 키 랭킹**: 요청·토큰 기준(usage_rollup) — 사용자별 사용량 추적.
- **사용량 추세 + forecast**: 시간 버킷 추세에 선형회귀 외삽 + 95% 예측 밴드.
- **GPU 3단 드릴다운**: 노드 LED → GPU 테이블 → per-GPU 시계열(util/mem/temp/power) + **유휴 할당 갭 KPI**(VRAM 점유·연산 유휴 GPU 수). MIG 미파티션은 사실대로 표시.
- **엔진 파이프라인 분해**: 평균 요청을 전처리→라우팅→큐→prefill(TTFT)→decode→전송으로 색분할(Tree/Waterfall 토글) — 트래픽 화면.
- **모델 운영 메트릭**: 카드에 tok/s·TTFT·컨텍스트·요구 GPU·서빙 패턴(agg/disagg) — 모델 화면.
- **응답별 메트릭 인라인**: 플레이그라운드 응답마다 지연·tok/s·입출력 토큰 칩.
- **키 예산 하드캡**: 일 토큰 예산(tpd) 초과 시 자동 **429** 차단 + 인앱 진행 게이지·경고 임계 + 추정 비용.
- **증적 status/latency**: 가드레일 증적에 HTTP status(차단 403/통과 200)·판정 지연 컬럼(SIEM 표준).
- **엔드포인트 실시간 로그**: 배포 Pod 로그 tail(컴포넌트 필터·자동 새로고침).

## 5. 안전장치 / 거버넌스

| 항목 | 내용 |
|---|---|
| 가드레일 선통과 | 모든 추론은 Semantic Router 분류 후 정책에 따라 차단/통과 |
| 증적 불변성 | `guard_audit`(ClickHouse) + MinIO WORM(Object Lock)로 변조 불가 보존(기본 365일) |
| 비식별화 | 증적 `user_ref` 솔트 해시 |
| 운영 리소스 보호 | 엔드포인트 삭제는 `fabrix.managed-by` 라벨 CR만, 보호 네임스페이스 거부 |
| 생성 안전 | 엔드포인트 생성은 기본 dry-run, `apply=true` 명시 시에만 실제 적용 |
| 비밀 보호 | 자격증명/비밀은 k8s Secret, 조회 시 마스킹, 평문 미반환 |
| 쿼터·귀속 | `x-api-key-id` 로 RPM/TPD 쿼터 및 부서/앱 귀속 |
