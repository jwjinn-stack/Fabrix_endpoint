### v0.16.0 — 2026-06-19 · P4 🟠 백엔드 데이터 보강 9항목 (관제 깊이 강화)
- **배경**: v0.14.0에서 P4 🟢(프론트만) 항목은 완료, 🟠(백엔드 데이터 보강 필요) 항목만 잔여였음. 이번 라운드에 9개 전부 **실데이터로 구현**(idle 시 0, 트래픽 발생 시 실측). QA 평균 9.3, console error 0.
- **🧭 IA 정리(사용자 피드백 반영)**: 추론 지연 분해·엔진 스케줄러·토큰 분해·Top5 랭킹은 **관제(overview)가 아니라 사용량(Usage) 리포트로 배치**. 관제는 한눈에 보는 글랜스 화면(KPI 4카드+분포+시계열+알람, **스크롤 없이 1화면**, scrollHeight=viewport)으로 유지하고, 분석 심화 지표는 사용량으로 모음. KPI 카드가 이미 TTFT p95·running/waiting 요약을 포함하므로 관제는 정보 손실 없음.
- ✅ **P4-1 추론 지연 3분할**: `LatencyBreakdown`(TTFT/TPOT/E2E × p50/p95/p99) + `SchedulerState`(running/waiting/queue p95/KV 캐시) + `TokenBreakdown`(입력/캐시/출력) → `overview` 확장 + `timeseries`에 TPOT/E2E/running/waiting 추가. live=dynamo_frontend 히스토그램·vllm:num_requests·request_plane_queue·cached_tokens 실측. 프론트 `LatencyPanel`(게이지)·스케줄러 카드·토큰 스택바. **검증: TTFT 97/TPOT 51/E2E 1900ms 실측.**
- ✅ **P4-1 Top5 엔드포인트/API키 랭킹**: usage_rollup(model·api_key 축) 조인 → `RankCard`. 키는 store ListKeys 로 표시명 해석. **검증: gemma 12요청, key_rb01/wm01 실데이터.**
- ✅ **P4-2 GPU 3단 드릴다운**: `GET /gpu/timeseries?uuid=`(per-GPU util/mem/temp/power 60분 시계열) → 행 클릭 시 우측 SlidePanel 스파크라인. **유휴 할당 갭 KPI**(VRAM 50%+·util<10% = 5/15 실측, Run:ai 패턴). MIG=미파티션(GPU_I_PROFILE 라벨 부재) → **정직하게 "전체 GPU 모드" 표시**(슬라이스 bargauge 대신 GR_ENGINE 효율). ※실 MIG 파티션은 하드웨어 미구성 = MAY.
- ✅ **P4-3 엔진 파이프라인 분해**: `GET /proxy/pipeline`(전처리→라우팅→큐→prefill(TTFT)→decode→전송, dynamo_frontend_stage_duration·request_plane_queue·TTFT·request_duration 실측) + Tree↔Waterfall 토글. **검증: 합계 483ms(prefill 52.8/decode 377.8) 실측.** ※개별 분산 스팬 트리는 victoria-traces 미수집 = MAY.
- ✅ **P4-4 사용량 forecast**: `GET /usage/trend`(usage_rollup 시간버킷 추세) + 프론트 최소제곱 선형회귀 외삽 + 95% 예측 밴드 + 요청/토큰 토글.
- ✅ **P4-5 예산 폼+하드캡**: 키 발급 모달에 **Portkey식 예산 폼**(rpm·일 토큰 예산·경고 임계 %·리셋 주기) + **인앱 예산 진행 게이지**(오늘 누적/tpd, 경고 임계 마커) + 추정 비용(자가호스팅 단가). **일 토큰 예산 하드캡 enforcement 신규**(quota 인메모리 일일 카운터 → tpd 초과 시 429). **검증: tpd=10 키 → 2차 요청 HTTP 429.** ※경고 임계는 인메모리(앱 DB 롤이 api_key owner 아님 → 컬럼 ALTER 권한 거부 42501. 영속화는 owner 마이그레이션 후속).
- ✅ **P4-6 모델 운영 메트릭**: `GET /models/metrics`(카탈로그 메타 + dynamo by-model 실측: tok/s=1000/TPOT·TTFT·E2E·컨텍스트·요구 GPU·agg/disagg) → 모델 카드에 칩 조인(서빙 중인 모델만, 미배포는 배지). ContextWindow·pattern 실데이터.
- ✅ **P4-7 응답별 메트릭 인라인**: 플레이그라운드 응답마다 지연·tok/s·입력/출력 토큰 칩 + 비교 모달 토큰 컬럼. ※TTFT는 비스트리밍이라 미표시(스트리밍 도입 후) = MAY.
- ✅ **P4-8 엔드포인트 Validate+로그**: 위저드 dry-run 검증 결과 강화(매니페스트 토글) + `GET /endpoints/{ns}/{name}/logs`(kubectl 파드 로그 tail, 컴포넌트 필터·자동 5s 새로고침) 실시간 로그 팝업. **검증: gemma 프론트엔드 실 로그.**
- ✅ **P4-9 증적 status/latency**: guard_audit 에 `http_status`(차단=403/통과=200)·`latency_ms` 컬럼 **비파괴 ALTER(ADD COLUMN IF NOT EXISTS, 우리 소유 증적 테이블)** + 적재/조회/스키마 반영. 증적 테이블에 SIEM 표준 컬럼. **검증: 차단 403·통과 200 실데이터.**
- 🐞 **QA 중 발견·수정**: ListKeys 500(`alert_threshold` 컬럼 추가 시도가 api_key owner 권한 부재로 ALTER 거부 → SELECT 깨짐). → api_key DDL 의존 제거, 경고 임계를 quota 인메모리로 전환(DDL 권한 불필요). 키 화면 복구.
