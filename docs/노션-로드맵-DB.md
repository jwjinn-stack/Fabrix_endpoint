# FABRIX1.x — 제품 기능 로드맵 (Notion 데이터베이스용)

> **용도**: "어떤 기능을 / 어디까지 / 어느 우선순위로 / 개발할 것이고 했는지"를 한 DB로 정리.
> 기존 Notion `Fabrix1.x Monthly Product Features Release Roadmap` 스키마(Features Back-log · Category · Priority · 고객요구 · 상태 · 담당자 · 월)에 맞춤.
> 근거: [기능-현황-로드맵.md](기능-현황-로드맵.md) · [기능-명세.md](기능-명세.md) · 코드 `v0.16.0`(2026-06-19).

## 사용법
1. 아래 **표를 Notion 페이지에 붙여넣으면** 데이터베이스로 변환됩니다(또는 §끝 CSV 블록을 `.csv` 저장 후 Import).
2. `상태`는 **실제 개발 상태**(✅완료/🚧진행중/⏳계획/⛔대기·보류) 기준입니다. 기존 Notion 의 "시작 전"은 계획 시점 값이라 본 표의 실상태로 갱신 권장.
3. `고객요구`는 삼성증권 미팅 요구 **테마**로 정리했습니다 — 기존 R1~R11 코드 체계가 있으면 매핑해 주세요(매핑 가이드 §끝).

- **Category**: 관제 · 사용량 · 가드레일 · 모델 · 키앱 · 플랫폼 · GPU · 인프라

- **상태**: ✅ 완료 · 🚧 진행중 · ⏳ 계획(Phase2) · ⛔ 대기/보류(외부 승인·인프라)

## 고객요구 R# 매핑 가이드
기존 Notion 의 `고객요구` 열은 R1~R11 코드를 씁니다. 본 표의 *테마*를 아래처럼 매핑하시면 됩니다(예시 — 실제 R 정의에 맞춰 조정):
- 모델성능 / 서비스모니터링 / 사용량추적 → 삼성증권 17일 미팅 3대 요구
- 거버넌스 → 가드레일·증적·WORM 관련 R
- 모델수명주기 / 검증 / 운영관리 / 관제 → 기 개발 기반 기능 R
