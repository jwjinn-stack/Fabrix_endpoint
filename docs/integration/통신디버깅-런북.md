# 통신 디버깅 런북 — 쿠버네티스 배포 점검

> `연동 상태`(`/diagnostics`) 화면을 보고 **이 Pod 에서 외부 의존성 통신이 제대로 되는지** 판단하고,
> 안 되면 **무엇을 고쳐야 하는지**를 엔지니어 관점에서 정리한 런북.
> 화면은 능동 프로브(read-only) 결과 + 단계 타이밍(DNS·TCP·TLS·서버) + 실패 원인 분류 + 파드 네트워크/설정 점검을 한 화면에 모은다.

---

## 0. 배포 직후 점검 순서 (위 → 아래)

1. **상단 "네트워크·설정 점검" 패널**부터 본다 — 프로브가 실패하기 *전에* 설정 오류를 잡는다.
   - `인클러스터` 배지가 켜졌나? (in-cluster API + serviceaccount 토큰)
   - `DNS` 가 kube-dns(예 `10.96.0.10`)인가? `search` 도메인에 네임스페이스가 있나?
   - 해석 테이블에서 각 의존성이 `scheme://host:port → IP` 로 **해석되는가**? (IP 가 비어 있으면 이름 해석 실패)
   - `프록시 경고`(빨간 줄)가 있나? → 인클러스터 트래픽이 프록시로 새는 중.
2. **외부 의존성 카드** — 빨강(실패)/주황(지연) 카드의 **실패 원인 배지**를 본다(아래 §1 표).
3. 카드의 **"통신 상세"** 펼치기 → 연결 IP·단계(ms)·TLS 인증서·원문 에러 확인.
4. 막히면 §2 의 `kubectl` 명령으로 클러스터 측 사실을 확인한다.

---

## 1. 실패 원인(FailKind) → 의미 → 조치

화면 카드의 빨간/주황 배지 = 백엔드 `diag.classify()` 분류. **종류마다 고칠 곳이 다르다.**

| 배지 | 무슨 일 | 1차 의심 | 확인/조치 |
|---|---|---|---|
| **이름 해석 실패** `dns_fail` | 호스트→IP 변환 자체 실패 | 서비스명 오타 · 네임스페이스(search domain) · CoreDNS | env 의 host 철자, FQDN(`svc.ns.svc.cluster.local`), CoreDNS Pod 상태 |
| **연결 거부** `conn_refused` | TCP 핸드셰이크 거부 | 포트 틀림 · 대상 Pod 미기동 · NetworkPolicy egress 차단 | Service/Endpoints 존재, 포트 일치, egress 정책 |
| **TLS/인증서 오류** `tls_fail` | TLS 핸드셰이크 실패 | CA 불신뢰 · SNI · 인증서 만료 · 프록시 MITM | "통신 상세"의 TLS 발급자/만료일, CA 번들, 프록시 |
| **인증 실패** `auth_fail` | 도달했으나 401/403 | 키/시크릿 틀림 · RBAC 권한 부족 | Secret 값, ServiceAccount Role/RoleBinding |
| **시간 초과** `timeout` | 응답 없음 | 방화벽 drop · 과부하 · 잘못된 IP 로 라우팅 | 단계 타이밍에서 어디서 멈췄나, 방화벽/NetworkPolicy |
| **비정상 응답** `bad_status` | 도달했으나 4xx/5xx | 업스트림 서비스 자체 이상 | 대상 Pod 로그, 업스트림 헬스 |
| **미구성** (회색) | env 미설정 → 폴백 동작 | 의도된 미구성인가? | 필요하면 `설정 → 연동`에서 구성 |

### 단계 타이밍으로 위치 좁히기
카드의 막대(DNS·TCP·TLS·서버)는 **어디서 시간을 쓰는지**를 보여준다:
- DNS 가 길다(100ms+) → CoreDNS/네임스페이스 검색 경로 문제.
- TCP 까지 가고 멈춤 → 방화벽이 SYN 이후 drop(흔한 사내망 함정).
- TLS 가 길거나(300ms+) 실패 → 인증서/프록시 MITM.
- 서버(TTFB)만 길다 → 네트워크는 정상, **업스트림이 느림**.

## 2. 클러스터 측 확인 명령 (화면 → kubectl)

화면에서 원인을 좁힌 뒤, 같은 Pod 안/클러스터에서 사실을 확인한다.

```bash
# 0) 진단 API 직접 호출(심층 진단 포함)
kubectl exec deploy/fabrix-endpoint -- wget -qO- localhost:8080/api/v1/diagnostics?verbose=1 | jq .

# 1) dns_fail — 이름 해석
kubectl exec deploy/fabrix-endpoint -- nslookup harbor-core.fabrix.svc.cluster.local
kubectl get pods -n kube-system -l k8s-app=kube-dns          # CoreDNS 살아있나
kubectl get svc <dep-svc>; kubectl get endpoints <dep-svc>   # Endpoints 비면 selector 불일치

# 2) conn_refused / timeout — 포트·egress
kubectl exec deploy/fabrix-endpoint -- nc -zv <host> <port>  # TCP 도달성
kubectl get networkpolicy -A                                  # egress 차단 정책
#   → default-deny 환경이면 kube-dns(UDP/TCP 53) + 대상 포트 egress 를 명시 허용해야 함

# 3) tls_fail — 인증서
kubectl exec deploy/fabrix-endpoint -- \
  sh -c 'echo | openssl s_client -connect <host>:443 -servername <host> 2>/dev/null | openssl x509 -noout -issuer -dates'

# 4) auth_fail — 시크릿·RBAC
kubectl get secret fabrix-secrets -o jsonpath='{.data}' | jq 'keys'   # 키 존재(값 노출 X)
kubectl auth can-i get pods --as=system:serviceaccount:<ns>:<sa>      # K8s 의존성 RBAC

# 5) 프록시 함정 — 인클러스터인데 프록시 경유
kubectl exec deploy/fabrix-endpoint -- env | grep -iE 'proxy'
#   → NO_PROXY 에 .svc,.cluster.local,<클러스터 CIDR> 추가
```

---

## 3. 자주 나오는 함정 (사내망 배포)

| 증상 | 원인 | 고침 |
|---|---|---|
| 모든 의존성 `dns_fail` | resolv.conf 가 kube-dns 가 아님 / dnsPolicy 오설정 | Pod `dnsPolicy: ClusterFirst` 확인 |
| 일부만 `dns_fail` | 단일 라벨 호스트인데 네임스페이스 다름 | FQDN(`svc.ns`)로 env 수정 |
| `conn_refused`인데 Pod 는 살아있음 | NetworkPolicy egress default-deny | 대상 포트 egress 허용 규칙 추가 |
| `tls_fail` 발급자가 사내 CA | CA 번들 미주입 | CA configmap 마운트 / `SSL_CERT_FILE` |
| 인클러스터 호스트가 `timeout` | 프록시가 `.svc` 트래픽까지 가로챔 | `NO_PROXY` 에 클러스터 도메인/CIDR 추가 |
| TLS 만료 D-30 이내(상세에서 빨강) | 인증서 갱신 누락 | cert-manager/수동 갱신 |

---

## 4. 화면 데이터가 어디서 오나 (참고)

- 능동 프로브: 각 클라이언트의 `Probe(ctx)` (read-only 1회). `ctx` 에 `httptrace` 주입 → 단계 타이밍/TLS 자동 수집.
- 네트워크 점검: `diag.BuildNetwork()` — `net.DefaultResolver.LookupHost`, `/etc/resolv.conf`, serviceaccount 토큰, `*_PROXY` env.
- 이력(sparkline): 인메모리 ring buffer(Pod 당 최근 50회, 재시작 시 초기화).
- 자격증명(비밀번호/키)은 **절대 표시하지 않음** (`diag.Redact()` + 프록시/URL 마스킹).
