## TL;DR — 따라잡을 5 + 능가할 3

### 따라잡을 것 (경쟁사가 이미 잘함 → 우리 백로그로)
1. **키 쿼터·예산(budget) 강제 + 키별 사용량 집계** — LiteLLM/Portkey/Nutanix 모두 키·팀·앱별 rpm/tpd·예산을 *강제*하고 초과 시 차단. 우리는 발급/회수/마스킹까지(✅)지만 쿼터 *적용·집계*는 ⏳.
2. **풍부한 가드레일 카탈로그(40~50종) + 정책을 UI에서 토글** — Portkey 50+, Kong 시맨틱 prompt guard, NeMo 5종 rails. 우리는 Semantic Router(PII) 1축 + 증적 파이프라인이 강점이나 "정책 카탈로그/토글 UX"는 부재.
3. **멀티모델 비교 플레이그라운드 + View code 스니펫(curl/python/js)** — Langfuse/Portkey/Databricks 표준. 우리 플레이그라운드는 단일모델(✅), 멀티비교·코드스니펫은 ⏳.
4. **프롬프트/평가(LLM-as-judge·데이터셋) 관리** — Langfuse·Databricks의 핵심 차별. 폐쇄망 증권사에서도 "응답 품질 회귀 테스트" 수요 큼. 우리는 미착수.
5. **트레이스 단위 관측(요청→리트리벌→툴 호출 타이밍)** — Langfuse/Helicone/Databricks Inference Table. 우리는 메트릭(vmselect)·집계 중심, *개별 요청 트레이스 뷰(4-5)*는 ⏳.

### 능가할 것 (FABRIX 차별 — 경쟁사 약점)
1. **증권사 온프렘/폐쇄망(air-gapped) + 가드레일 증적 WORM** — Lakera/Portkey/Databricks는 SaaS·VPC 하이브리드가 본진. 다크사이트 풀스택(게이트웨이+가드레일+관측+증적 보존)을 *국내 규제(전자금융·망분리)* 기준으로 통합 제공하는 곳은 드묾. NAI가 가장 근접하나 증적 WORM·한국 PII는 약함.
2. **부서·앱·키·모델 4축 귀속(사내 DB identity-broker 연동)** — 대부분 키/팀 2축. 사내 인사 DB와 소급 enrich해 "누가(직원/부서)·어느 앱이 GPU를 얼마나" 까지 귀속하는 건 SI 맞춤 강점.
3. **Dynamo/MIG 네이티브 GPU 관제(DCGM) + MIG 효율 스코어** — AI 게이트웨이/관측 제품군은 GPU 물리 계층(MIG 분할 효율, disagg/NIXL)까지 못 본다. NIM/Run:ai가 GPU엔 강하나 거버넌스·증적 레이어는 약함. 우리는 둘을 한 화면에.

**FABRIX 적용:**
- *따라잡을 것*: Lakera의 **탐지 카테고리 폭**(인젝션·탈옥·시크릿·유해성·악성링크)과 *지연 목표(<50ms)* 를 우리 Semantic Router(ext_proc :50051) 경로의 SLO로 삼기. NeMo의 **rails 5종 분류**를 우리 정책 모델 설계 어휘로 차용(input/output/retrieval/dialog).
- *능가할 것*: Lakera는 SaaS·폐쇄망 약함, NeMo는 *증적 보존·귀속·관제 UI* 없음(라이브러리). FABRIX는 **온프렘 가드레일 실행(SR) + 차단/PII/탈옥 증적의 WORM 보존(4-3) + 한국어 PII(주민/계좌/여권) + trace_id·정책버전·마스킹샘플까지 보이는 증적 뷰** — 이게 증권사 규제 대응의 핵심 차별. (NeMo를 *대체*가 아니라 *내장 엔진 후보*로 채택 가능 — 추측.)

## 능가 전략 (FABRIX 차별 축 강화)

1. **"증권사 폐쇄망 컴플라이언스 OS"로 포지셔닝**: 게이트웨이+가드레일+관측+증적을 *한 번에*, 망분리·다크사이트 가정. 경쟁사는 SaaS/하이브리드가 본진이라 air-gap 풀스택에서 깨진다(Lakera/Databricks/Cloudflare 탈락). NAI가 유일 근접 → **증적 WORM·한국 PII·Dynamo/MIG** 3축으로 차별.
2. **귀속(Attribution)을 1급 시민으로**: 부서·앱·키·모델 4축 + 사내 인사 DB enrich(identity-broker). "누가(직원)·어느 앱이 GPU를 얼마나·어떤 가드레일에 걸렸나"를 단일 질의로. 경쟁사 최대치는 팀/유저 2축.
3. **가드레일 증적의 불변성(WORM)**: 차단/PII/탈옥 이벤트를 trace_id·정책버전·마스킹샘플과 함께 ClickHouse→ObjectScale WORM. 규제 감사 대응이 경쟁 제품의 사각.
4. **GPU 물리계층까지 내려가는 관제**: DCGM 메트릭 + MIG 분할 효율 스코어 + disagg/NIXL. 거버넌스 제품군은 못 보고, GPU 제품군(NIM/Run:ai)은 거버넌스를 못 본다 — 교집합이 우리.
5. **엔진은 채택, 레이어로 차별**: vLLM/Dynamo/NIM/NeMo는 *대체 대상이 아니라 내장/연동 후보*. "우리는 그 위의 거버넌스·관제·증적 레이어"라는 메시지 일관화(추론 raw 성능 경쟁은 지양).
