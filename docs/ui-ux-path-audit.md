## 0. 먼저 — 전체 그림

### 0.1 기반은 이미 성숙하다 (P0 대부분 충족)
리서치 리포트의 P0 권고 상당수가 **이미 구현돼 있다.** "없는 걸 추가"가 아니라 "있는 걸 페이지별로 일관 적용"이 과제다.

| 기반 항목 | 상태 | 근거 |
|---|---|---|
| 디자인 토큰(색·간격·타이포 스케일) | ✅ | `index.css` `:root` — 8pt 그리드, 타이포 스케일, 시맨틱 색 |
| `:focus-visible` 포커스링 (WCAG 2.4.7) | ✅ | `index.css:97` |
| `prefers-reduced-motion` | ✅ | `index.css:107` |
| `tabular-nums` | ✅ | `.metric .num`, `.usage-table .num` 등 |
| aria-live 스켈레톤 | ✅ (단, **적용 페이지 편차**) | `Skeleton.tsx` SkeletonCards/Rows |
| SlidePanel 드릴다운 통일(중앙 모달 폐기) | ✅ | `SlidePanel.tsx` — 전 목록 상세 통일 |
| ⌘K 커맨드 팔레트 | ✅ | `CommandPalette.tsx` |
| 시맨틱 색 ≠ 액센트 분리 | ✅ | green/amber/red/pink/teal 별도 토큰 |

### 0.2 ✅ 해결됨 — 액센트는 "설정에서 변경"하는 방식 (블루 기본 + 정합화 완료)
액센트를 고정 결정하지 않고 **설정 화면에서 바꾸는 방식**으로 확정. 이 기능은 이미 구현돼 있었다.

- `theme.tsx` — `BRAND_PRESETS`(스틸블루 기본 · **오렌지 `#fb6e00`** · 틸 · 인디고 · 슬레이트) + 커스텀 HEX 피커(`deriveBrand`로 strong/weak/lite 자동 파생) + `--primary` 계열 CSS 변수 오버라이드 + localStorage 영속. 라이트/다크 공통.
- `Settings.tsx`의 `BrandColorCard` → `/settings`에서 즉시 전환.

**정합화 작업(2026-06-29 적용):** 브랜드 색을 바꿔도 안 따라오던 하드코딩 누수 제거.
1. `index.css` — `--focus-ring: #4a86b8` → **`var(--primary)`** (포커스링이 액센트를 따라감). *진짜 버그였음.*
2. **제품 마크 ≠ UI 액센트 분리** — FABRIX 워드마크의 고정 오렌지를 `--brand-fabrix` 토큰으로(고객 액센트와 무관한 제품 고유색). `capabilities.tsx` 부팅 워드마크 적용.
3. `Layout.tsx` "관제 전용" 배지 — 하드코딩 오렌지 → 흰색-반투명(상단바가 어떤 액센트여도 가독). 차트는 이미 전부 `var(--primary)` 사용이라 자동 반영됨.

> **알려진 한계(별도 과제):** `applyBrand`가 다크 모드에서도 라이트 기준 `--primary-weak`(밝은 틴트)를 덮어써, 다크+커스텀 브랜드 조합 시 옅은 배경이 과하게 밝을 수 있음. 다크는 옵트인이라 우선순위 낮음.

**원칙:** 이 문서의 모든 "강조색" 권고는 액센트가 무엇이든 동일 — 핵심은 "한 색을 의미(action/active/positive)에만 절제해서" 쓰는 것.

### 0.3 다크 테마
`index.css:70` 에 `[data-theme="dark"]` 옵트인 존재(네온 배제, 블루 보존). 리서치의 "다크 금지"는 **다크를 기본값/유일값으로 강요하는 트렌드**에 대한 경고였다. **옵트인 다크는 문제없음** — 단 기본은 라이트 유지 권장.

### 0.4 가장 공통된 갭 3가지
1. **스켈레톤 적용 편차** — Dashboard/Traces/Sessions만 스켈레톤, **Usage·Gpu·Traffic·Guard·Settings 등은 텍스트 "로딩 중"만**. → 체감속도 손해.
2. **기준선(delta/baseline) 누락** — 일부 KPI에 전기간 대비 변화율·정상범위 표기 없음.
3. **고위험 액션 마찰 부재** — 엔드포인트 삭제·키 회수·권한 상향이 **확인 다이얼로그 없이 즉시 실행**. (금융 엔터프라이즈에서 특히 위험)

## 1. 관측 / 대시보드 클러스터

### `/dashboard` — Dashboard.tsx
**현황** · Live API(15초 자동갱신) · SkeletonCards 로딩 ✅ · KPI 카드 3(트래픽/응답/가드레일) + GPU·MIG 토글 + 부서/앱 BarList 2 + TimeseriesChart(QPS/TTFT/차단) + 알람 · BarList 행→SlidePanel 드릴다운, 차트 드래그줌+키보드.

**갭**
- ❌ **"지금 정상?" 골든시그널 요약이 없다.** 관제 페이지인데 한눈에 적/녹 신호등이 없음.
- ⚠️ 정보 과부하 — 부서/앱 분포 2개가 항상 노출(요약용 대시보드엔 과함).
- ⚠️ TimeseriesChart 범례 3종이 한 줄 → 모바일 줄바꿈 깨짐, 차단 막대 시각적 약함.

**수정 (무엇을 → 어떻게)**
1. **골든시그널 요약 카드 신설** → KPI 그리드 위(`Dashboard.tsx` 상단)에 StatCard 1개: 성공률>99% AND TTFT p95 임계 AND 차단율<1% 의 AND 조건으로 green/amber/red 결정.
2. **분포도 접기** → GPU 토글과 동일 패턴으로 부서/앱 BarList를 기본 숨김 + "더 보기" 토글(`panels` localStorage 패턴 재사용).
3. **범례 반응형** → `TimeseriesChart.tsx`의 `.chart-legend`를 2열 grid + 모바일 축약.

**현황** · ClickHouse(라이브) + Langfuse(lazy) · 로딩=텍스트(**스켈레톤 ❌**) · 요약 카드(검사/차단/PII/Jailbreak) + 히스토그램 + 증적 테이블 + SlidePanel · decision/유형/신뢰도 배지(절제됨 ✅).

**갭** (리서치의 가드레일 핵심 "규칙→이유→통과조건"이 **부분만** 충족)
- ⚠️ 규칙명은 있으나 "왜 트리거됐는지, 무엇을 바꾸면 통과되는지"가 약함.
- ⚠️ 정책 버전은 있으나 적용 시점·이전 대비 차이 없음.
- ⚠️ 마스킹 적용 표시는 있으나 어떤 PII 유형인지 불명확.

**수정**
1. **3요소 명시** → SlidePanel verdict 박스: "Jailbreak 88% → 정책 '차단' 설정됨 · [정책 변경]" 식으로 규칙·통과조건 연결. **(가드레일 정체성 핵심)**
2. **정책 버전 추적** → "정책 v{n} 적용(배포일시) · [변경 이력]" 링크.
3. **마스킹 유형 표기** → "마스킹된 유형: 주민번호, 전화" (원문 미포함, 정책만).

**갭**

- ❌ 권한 상향(user→admin)·삭제에 **확인 없음**.

**수정**

**현황** · live(심층 진단 옵션) · 요약 상태(도달/구성/점검필요/전체) + 네트워크 점검(DNS/인클러스터/프록시) + 의존성 타일(상태 배지 + 단계 타이밍 막대 + sparkline + 원인/조치 힌트 + 확장 상세 + 지금 테스트) + InspectDrawer(DevTools식 탭).
→ **연동 진단의 능동성/깊이는 이미 우수.**

**갭**

2. **조치 버튼화** → 힌트를 `[설정 → 연동 고치기]` 링크로.
3. **raw 네트워크 로그** → `<details>` 접힘으로 DNS/TCP/TLS 이벤트.
